## 개요

사용자가 자동 뉴스 수집 대상이 되는 관심 종목을 등록·수정·삭제·조회한다.
등록된 종목 목록은 n8n 자동 수집 파이프라인에서 참조된다.

---

## 사용자 스토리

- 나는 삼성전자, SK하이닉스 등 관심 있는 종목을 등록하여 해당 종목 뉴스만 수신하고 싶다.
- 더 이상 관심 없는 종목은 삭제하여 불필요한 뉴스를 줄이고 싶다.
- 등록된 종목 목록을 한눈에 확인하고 싶다.

## 주요 흐름

### 정상 흐름 - 종목 등록

```
1. [+ 종목 추가] 버튼 클릭
2. 추가 모달 열림
3. 종목 코드, 종목명 입력 (필수)
4. 섹터, 거래소 입력 (선택)
5. [저장] 클릭
6. POST /api/v1/stocks 호출
7. 성공 시 모달 닫힘 + 목록 갱신 + "종목이 등록되었습니다." 토스트
```

### 정상 흐름 - 종목 삭제

```
1. 목록에서 🗑️ 클릭
2. "삼성전자 종목을 삭제하시겠습니까?" 확인 다이얼로그
3. [삭제] 확인
4. DELETE /api/v1/stocks/{id} 호출
5. 목록에서 즉시 제거 + "종목이 삭제되었습니다." 토스트
```

### 예외 처리

| 상황 | 처리 방법 |
|------|---------|
| ticker 중복 (409) | "이미 등록된 종목 코드입니다." 모달 내 인라인 에러 |
| 유효성 검증 실패 (400) | 각 필드 아래 에러 메시지 표시 |
| 서버 오류 (500) | "일시적인 오류가 발생했습니다." 토스트 |

## 수락 기준 (Acceptance Criteria)

### BE (✅ Week 1 완료)
- [x] 종목 코드, 종목명을 입력하여 종목을 등록할 수 있다. (POST /api/v1/stocks)
- [x] 동일한 종목 코드로 중복 등록 시 409 에러가 반환된다.
- [x] 등록된 종목을 수정할 수 있다. (PUT /api/v1/stocks/{id})
- [x] 종목 삭제 시 소프트 삭제 처리된다. (DELETE /api/v1/stocks/{id}, isActive=false)

### FE (⏳ Week 3 예정)
- [ ] 종목 추가 모달이 표시된다.
- [ ] 종목 삭제 시 확인 다이얼로그가 표시된다.
- [ ] 삭제된 종목은 목록에서 즉시 사라진다.
- [ ] 빈 목록일 때 "등록된 종목이 없습니다." 빈 상태 메시지가 표시된다.
