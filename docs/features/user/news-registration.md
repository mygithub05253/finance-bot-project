## 개요

사용자가 뉴스 URL을 직접 입력하면 ai-service가 크롤링 → Claude API 분류/요약 → 아카이브 저장까지 자동 처리한다. 목표 처리 시간은 3초 이내.

---

## 사용자 스토리

- SNS나 다른 앱에서 흥미로운 금융 뉴스를 발견하면 URL만 붙여넣어서 아카이브에 저장하고 싶다.
- 저장 결과(종목, 카테고리, 요약)를 즉시 확인하고 싶다.
- 이미 등록한 URL을 다시 등록할 경우 알림을 받고 싶다.

## 주요 흐름

### 정상 흐름

```
1. URL 입력 필드에 뉴스 URL 붙여넣기
2. [등록] 버튼 클릭 (또는 Enter)
3. 로딩 스피너 표시 ("AI가 분석 중입니다...")
4. POST /ai/v1/news/register 호출
5. 3초 이내 응답 수신
6. 결과 카드 표시:
   - 종목명 + 종목 코드
   - 카테고리 배지
   - 감성 배지
   - AI 요약 텍스트
   - 키워드 태그
7. URL 입력 필드 초기화 (다음 URL 바로 입력 가능)
```

### 예외 처리

| 상황 | 에러 코드 | 사용자 메시지 |
|------|---------|-------------|
| 유효하지 않은 URL 형식 | 클라이언트 검증 | "올바른 URL 형식을 입력해주세요." (인라인) |
| 중복 URL | `DUPLICATE_URL` (409) | "이미 등록된 URL입니다." 토스트 (이전 등록일 표시) |
| 크롤링 실패 | `CRAWL_FAILED` (422) | "페이지에 접근할 수 없습니다. (로그인 필요 또는 차단)" |
| Claude API 오류 | `CLASSIFY_FAILED` (500) | "AI 분류 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요." |
| 3초 초과 | 클라이언트 타임아웃 | 로딩 메시지를 "조금 더 걸리고 있습니다..." 로 변경 (최대 10초) |

## Claude 분류 프롬프트 설계 (참고)

```
역할: 너는 한국 금융 뉴스를 분류하는 전문 애널리스트야.

다음 뉴스를 분석하여 JSON 형식으로 응답해줘:

뉴스 제목: {title}
뉴스 내용: {content}

관심 종목 목록: {stockList}  // [{ticker, name}] 형태

응답 형식:
{
  "stockTicker": "005930",  // 관련 종목 코드 (없으면 null)
  "category": "실적",       // 실적/규제/M&A/인사/기타 중 하나
  "summary": "3-5줄 한국어 요약",
  "sentiment": "POSITIVE",  // POSITIVE/NEUTRAL/NEGATIVE
  "keywords": ["키워드1", "키워드2"]  // 최대 5개
}
```
