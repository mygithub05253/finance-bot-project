## 개요

자동 수집 및 수동 등록된 뉴스를 날짜·종목·카테고리별로 필터링하여 카드 형태로 조회한다.
대시보드 메인 화면으로, 뉴스 히스토리를 아카이브로 관리하는 핵심 기능.

---

## 사용자 스토리

- 오늘 수집된 금융 뉴스를 카드 형식으로 한눈에 확인하고 싶다.
- 삼성전자 관련 뉴스만 모아서 보고 싶다.
- 지난달 수집된 M&A 관련 뉴스를 검색하고 싶다.

### 뉴스 카드 구성 요소
- 종목명 + 종목 코드
- 카테고리 배지 (실적 / 규제 / M&A / 인사 / 기타)
- 감성 배지 (POSITIVE: 초록 / NEUTRAL: 회색 / NEGATIVE: 빨강)
- AI 요약 텍스트 (3-5줄)
- 키워드 태그
- 수집 방식 배지 (AUTO / MANUAL)
- 원문 링크

## 주요 흐름

### 정상 흐름 - 목록 조회

```
1. 페이지 접근 시 오늘 날짜 기본값으로 GET /api/v1/news?date=오늘 호출
2. 서버 컴포넌트에서 뉴스 목록 + 종목 목록 병렬 fetch
3. 뉴스 카드 그리드 렌더링 (2열, 모바일 1열)
4. 필터 변경 시 URL 쿼리 파라미터 업데이트 → 서버 재요청
```

### 정상 흐름 - 필터 적용

```
1. 날짜 피커에서 날짜 선택
   → URL: /?date=2026-03-10
2. 종목 드롭다운에서 선택
   → URL: /?date=2026-03-10&stockId=1
3. 카테고리 선택
   → URL: /?date=2026-03-10&stockId=1&category=실적
4. 각 필터 변경마다 서버 재요청 (SSR)
```

### 예외 처리

| 상황 | 처리 방법 |
|------|---------|
| 조회 결과 없음 | "해당 조건의 뉴스가 없습니다." 빈 상태 메시지 + 일러스트 |
| 서버 오류 | Next.js error.tsx 에러 바운더리 |
| 네트워크 오류 | "데이터를 불러올 수 없습니다. 새로고침 해주세요." |

## 수락 기준 (Acceptance Criteria)

- [ ] 첫 접속 시 오늘 날짜 기준 뉴스 목록이 표시된다.
- [ ] 날짜·종목·카테고리 필터를 조합하여 뉴스를 필터링할 수 있다.
- [ ] 감성 분석 결과가 색상 배지로 직관적으로 표시된다.
- [ ] 키워드 태그를 클릭하면 해당 키워드로 검색된다. (v1.1 이후)
- [ ] 원문 링크를 클릭하면 새 탭에서 원문 페이지가 열린다.
- [ ] 페이지당 20건 표시, 페이지네이션 동작이 정상이다.
- [ ] 모바일 화면에서 1열, 데스크탑에서 2열 레이아웃이 적용된다.
